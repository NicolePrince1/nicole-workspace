# Add Template

> Save a project as a reusable starting point.

A **template** is a saved project scaffold — pages, widgets, date range, and settings — that you can apply to any new project. Templates save you from rebuilding the same report structure for every new client.

## Create from an existing project

<Steps>
  <Step title="Open the source project">
    The project whose layout you want to reuse.
  </Step>

  <Step title="Click Save as Template">
    From the project's three-dot menu. Oviond copies the pages and widgets (minus the data values) into a new template.
  </Step>

  <Step title="Name and describe it">
    Give it a clear name (e.g. *Paid Social Monthly*) and an optional description so your team knows when to pick it.
  </Step>

  <Step title="Optional: upload a thumbnail">
    A thumbnail helps teammates find the right template at a glance.
  </Step>

  <Step title="Save">
    The template appears in the [Templates library](/templates/list) and is available to every new project.
  </Step>
</Steps>

## What gets saved

* Every page and its layout
* Every widget's type, data-source key, metric, dimension, and display settings
* Project-level date range and PDF export settings

## What doesn't

* Client-specific data connections — these are re-wired per project when the template is applied
* Live values — the template holds the structure, not the data
