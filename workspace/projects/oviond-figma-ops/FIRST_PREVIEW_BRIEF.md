# Oviond first Figma preview brief

## Objective
Create the first seed visuals for the Oviond design system so Chris can judge whether the direction feels premium, scalable, and recognizably Oviond before we build the Figma template library.

## Brand rules in play
- white-led compositions first
- restrained palette
- MAIN BLUE `#0000FF` as strongest accent
- secondary blues only as support
- abstract and feeling-led, not office stock imagery
- premium SaaS, spacious, credible, commercial
- blog covers should bias toward product-preview aesthetics
- avoid trusting in-image text

## First preview set
1. Website hero visual, wide, product-adjacent, elegant reporting UI emerging from white space
2. Blog cover, wide, layered report architecture with believable UI detail windows
3. Social / paid concept, square or portrait, chaos-to-clarity visual metaphor
4. Abstract brand texture / support visual for sections, banners, or website modules

## Goal of this batch
Not final production assets.

This batch is for:
- visual territory validation
- confirming the right level of abstraction vs product realism
- seeing whether Oviond should lean more product-preview or more atmospheric brand-led
- deciding what becomes the base Figma component/template family
