# Weekly Oviond Summary Spec

## Purpose

Give Chris a sharp Monday-morning briefing he can use immediately for the weekly team update.

## Output rules

- Open with a blunt 1-2 sentence verdict.
- No fluff, no vanity-metric parade.
- Say what changed, what matters, and what happens next.
- If data is noisy, say so plainly.
- Prefer bullets over walls of text.

## Standard structure

1. **Headline**
   - One blunt read on where Oviond stands right now.

2. **Scorecard vs goals**
   - Revenue / MRR trajectory
   - Retention / churn
   - US acquisition engine
   - Measurement quality
   - Lifecycle / automation rebuild
   - Customer truth / support signal
   - Mark each as: on track, mixed, or off track.

3. **What we achieved last week**
   - Real shipped changes, fixes, launches, or strategic conclusions.

4. **Where we are now**
   - Current commercial and channel state.
   - Include only the metrics that actually matter.

5. **Main priorities this week**
   - The 3-6 workstreams that deserve focus now.

6. **Risks / watchouts**
   - What could hurt growth, conversion, or confidence if ignored.

7. **Decisions needed from Chris**
   - Only include if there is a real choice or unblock needed.

8. **Team update talk track**
   - A short set of bullets Chris can lift straight into the meeting.

## Source priority

1. Stripe / backend truth
2. Ad-platform truth
3. Search Console
4. Segmented GA4 reads
5. Recent memory notes and weekly logs

## Interpretation rules

- Do not treat top-line GA4 conversions as trusted growth truth.
- Judge South Africa paid activity by quality and stickiness, not just cheap CPA.
- Favor decisions that improve retention, activation, and client-ready confidence.
- If a number is stale but still directionally useful, say that.

## Delivery rules

- Save each report to `memory/weekly-reports/YYYY-MM-DD.md`.
- Send the same report in chat and by email to `chris@oviond.com`.
- Email subject format: `Oviond Weekly Summary | YYYY-MM-DD`.
