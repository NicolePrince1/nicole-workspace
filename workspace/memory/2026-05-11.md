## OpenAI realtime voice update review

- Chris asked for a full review of OpenAI's May 7 voice-model update and how OpenClaw can use it for Nicole's voice workflow.
- OpenAI announced three Realtime API models: `gpt-realtime-2` for reasoning voice agents, `gpt-realtime-translate` for live translation, and `gpt-realtime-whisper` for streaming transcription.
- Current OpenClaw runtime inspected on this host: OpenClaw `2026.5.6`; TTS is enabled with `auto: inbound`, provider `openai`, model `gpt-4o-mini-tts`, voice `coral`, speed `1.25`; `tools.media.audio` is not explicitly configured.
- OpenClaw natively supports: outbound TTS via `messages.tts`; inbound batch audio transcription via `tools.media.audio`; Telegram native voice-note delivery; Control UI Talk via realtime voice; and Voice Call realtime voice/streaming transcription via plugin config.
- Current OpenClaw OpenAI realtime defaults in docs/source are still `gpt-realtime-1.5` for realtime voice and `gpt-4o-transcribe` / `whisper-1` paths for transcription, so adopting `gpt-realtime-2` or the new realtime transcription/translation models likely needs config override and live smoke testing, not blind assumption.
