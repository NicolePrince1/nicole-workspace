# 2026-03-28

## Attribution Hub build started

- Created a new Railway-targeted app at `projects/oviond-attribution-hub/`.
- The app is a Next.js + TypeScript internal tool that treats Oviond backend events as the source of truth, stores them in Postgres, and fans them out to Meta, GA4, Google Ads, and Stripe-aware flows.
- Current MVP includes:
  - canonical event schema and validation
  - Postgres event ledger + delivery-attempt log
  - ingest API with API key auth
  - Meta Conversions API adapter
  - GA4 Measurement Protocol adapter
  - Google Ads delivery scaffold / disabled-safe adapter
  - Stripe webhook ingestion
  - replay endpoint for re-dispatching stored events
  - protected dashboard login using `ADMIN_TOKEN`
  - Railway setup docs in the project README
- `npm run lint` and `npm run build` both passed after implementation.

## Attribution system direction confirmed

- Chris explicitly wants a proper Railway-hosted attribution app built, not just a tracking spec.
- Core measurement rule for Oviond going forward: the backend decides whether a conversion happened; GTM, GA4, Meta/Facebook Ads, and Google Ads are downstream observers; Stripe is the revenue source of truth.
- Canonical business events for the app and tracking model are:
  - `signup_started`
  - `email_verified`
  - `trial_started`
  - `subscription_started`
  - `invoice_paid`
  - `trial_expired`
  - `subscription_cancelled`

## Meta tracking finding

- The new Meta campaign `Remarketing USA - Free Trial Test` is materially better on top-of-funnel traffic than the older remarketing campaign, but actual free-trial conversion measurement is still broken/unreliable.
- Durable lesson: do not trust the current Meta `Customer Signs Up` / legacy custom conversion setup or GA4 `ads_conversion_signup` as the source of truth for actual trial starts.
