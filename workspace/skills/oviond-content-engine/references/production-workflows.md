# Production Workflows

## Default stance

Create from truth, then adapt, then schedule.
Do not start with calendar cosmetics.

## Weekly operating rhythm

### 1. Source review
At the start of the cycle, review:
- product updates
- support friction
- live acquisition lessons
- lifecycle or retention issues
- search demand
- customer objections
- anything Chris explicitly wants pushed this week

### 2. Pick the weekly content priority
Choose one of these as the lead theme:
- acquisition
- activation
- retention
- product proof
- category positioning
- feature or integration launch

### 3. Build the weekly package
A strong week usually contains:
- 1 core asset, often a blog or substantial LinkedIn post
- 2 to 4 derivative social posts
- 1 supporting visual or carousel if useful
- 1 repost or refreshed winner if justified

## Blog-first workflow

1. lock the audience and argument
2. write the working title
3. outline 3 to 5 sections
4. gather proof, screenshots, or examples
5. write the article in full
6. tighten headline and intro
7. extract derivative angles for LinkedIn, X, and Facebook
8. brief visuals if needed
9. prepare Postiz drafts or schedule plan

## Social-first workflow

Use this when the idea is timely or sharp enough that a full article is unnecessary.

1. pick one idea only
2. write 3 hook options
3. choose the strongest one
4. draft the native platform version first
5. adapt to secondary channels
6. attach visual only if it improves comprehension or stop power
7. queue in Postiz

## Feature launch workflow

1. define what changed
2. define why it matters to agencies
3. identify the right proof, screenshot, or demo angle
4. create one canonical launch message
5. adapt it for:
   - blog or changelog-style article if depth matters
   - LinkedIn product post
   - X launch post or short thread
   - Facebook update if worth syndicating
6. schedule supporting follow-ups if the feature is strategically important

## Repurposing workflow

When a strong asset already exists:
- pull out 3 to 5 atomic ideas
- map each idea to the best channel
- rewrite from scratch for that channel
- stagger publication so the same thought is not dumped everywhere at once

## Approval ladder

### Default
- draft the content
- schedule it only after review or explicit standing delegation

### Fast path
If Chris has clearly said to proceed on a defined topic or batch, you can:
- create the full set
- place it in Postiz as drafts or scheduled posts
- present the batch concisely

### Live publish
Only publish immediately when one of these is true:
- Chris explicitly asked for live posting now
- there is an established explicit autonomy rule for that channel and content class

## Quality gate

Before a piece ships, check:
- is the hook strong enough?
- is the claim true and specific?
- does it sound like Oviond, not generic SaaS mush?
- is there one main point?
- is the CTA clear?
- is the channel formatting native?
- if visualized, does the creative help rather than decorate?

## Failure patterns to kill

Reject drafts that are:
- generic
- multi-message and messy
- all setup, no point
- overloaded with features
- too timid to be memorable
- too hypey to be credible
- obviously copy-pasted across channels
