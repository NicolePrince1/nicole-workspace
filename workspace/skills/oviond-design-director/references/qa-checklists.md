# QA Checklists

Run the relevant checklist before approving output.

## Universal checklist

Ask:
1. Is the message obvious within 2 seconds?
2. Is there one clear focal point?
3. Does this feel like a credible SaaS brand, not prompt sludge?
4. Is the composition clean enough to survive resizing/cropping?
5. Is anything decorative but not useful?
6. Is the hierarchy strong enough without explanation?
7. Does it support the actual business goal?

Reject if:
- the idea is muddy
- the layout is crowded
- the model introduced weird artifacts
- it looks generic or off-brand
- it depends on tiny text to work
- the model mangled spelling or copy in a way that hurts credibility
- the composition should clearly have been text-free or overlay-led instead of forcing in-image copy

## Paid ad checklist

Ask:
- Does it stop the scroll?
- Is the value prop visible fast?
- Is the CTA zone clean?
- Is contrast strong enough for mobile?
- Would this still work if viewed for less than a second?

## Social / organic checklist

Ask:
- Is it engaging without looking like clickbait?
- Is the message singular?
- Does it look polished enough to represent the brand?
- Is it visually interesting without becoming noisy?

## Web visual checklist

Ask:
- Does it support, not fight, the page headline?
- Is there enough whitespace for layout integration?
- Does it build trust?
- Would this still look good on a modern SaaS page?

## Stat card checklist

Ask:
- Is the number the hero?
- Is the support copy short enough?
- Can the asset be understood instantly?

## Iteration rules

When something is off, name the issue precisely:
- too busy
- too generic
- weak focal point
- low contrast
- not enough brand character
- too much text
- not useful for placement
- too abstract

Then revise the prompt in that direction instead of random re-rolling.
