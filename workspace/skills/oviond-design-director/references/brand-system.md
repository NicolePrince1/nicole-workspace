# Oviond Brand System

Use this file as the governing visual reference for generated creative.

## Brand intent

Oviond should feel:
- clear
- capable
- commercially useful
- modern without being trendy for the sake of it
- simple enough for agencies to trust quickly
- premium enough to avoid looking cheap

This is not a loud, chaotic, hyper-playful brand.

## Product truth to preserve

Oviond is a reporting platform for marketing agencies. Core positioning themes:
- simple reporting
- white-label reporting
- easy setup
- low friction
- less clutter
- faster path to useful client reporting

Design should reinforce these truths.

## Visual personality

Bias toward:
- clean composition
- clear hierarchy
- breathing room
- restrained but confident color
- polished UI/product-adjacent aesthetics
- useful visual metaphors, not random symbolism

Avoid:
- noisy gradients everywhere
- generic startup blobs
- cluttered pseudo-dashboards
- overpacked comparison graphics
- cheesy productivity imagery
- overly corporate stock-photo scenes
- crypto-looking neon nonsense

## Composition guidance

Prefer:
- one dominant focal point
- a strong headline zone
- generous spacing
- obvious reading order
- layouts that can survive resizing

For most assets, use a hierarchy of:
1. primary message
2. proof/supporting visual
3. CTA or brand anchor

## Color and contrast

Until a stricter brand palette file exists, use this practical rule:
- keep the base clean and light or dark, not muddy
- use a restrained SaaS palette with one primary accent and one support accent
- prioritize legibility over novelty
- avoid rainbow palettes unless the asset specifically calls for them

If generating from scratch, keep palettes tight: usually 2-4 dominant colors plus neutrals.

## Typography guidance

If the image model must render text:
- keep text very short
- use simple, bold, high-legibility phrasing
- avoid long paragraphs in-image
- prefer headline-style text over sentence-style text

If production quality matters, treat text as a post-generation overlay whenever possible.

## Product/UI usage

When using dashboard-style imagery:
- keep charts simple and believable
- show order, not chaos
- imply reporting clarity, not data overload
- avoid tiny illegible widgets
- do not fake impossibly complex enterprise dashboards

## Channel posture

- **LinkedIn / organic social:** confident, useful, smart, lightly polished
- **Paid social ads:** fast message comprehension, stronger contrast, sharper hierarchy
- **Web / landing pages:** trust, product clarity, conversion support
- **Email / lifecycle graphics:** simple, lightweight, not visually heavy
- **Blog graphics:** clean supporting visuals, not clickbait thumbnails

## Quality bar

A good Oviond design should look like it belongs to a competent SaaS company serving agencies, not like a prompt demo from an image model community.
