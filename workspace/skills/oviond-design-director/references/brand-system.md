# Oviond Brand System

This file is the working brand CI reference for design tasks. It is based on the November 2024 Oviond brand refresh guide and should override earlier placeholder assumptions where they conflict.

## Source basis

Primary source: Oviond Brand Refresh (November 2024).

Known explicit source points captured from the guide:

- date: `November 2024`
- logo refresh section
- logo type font change to `INTER SEMIBOLD`
- logo comparison between refreshed and original logo
- multiple logo applications: `stacked`, `horizontal`, and `web`

- channel/application examples for `Web & Social`, `Facebook`, `LinkedIn`, and `YouTube`

## Core brand idea

The guide pushes Oviond away from literal business clichés and toward a broader, more ownable visual feeling.

Use these source ideas as non-negotiables:
- use **abstract and unspecific imagery**
- do **not** always focus on offices, meetings, and computers
- use aesthetics to trigger a **feeling** rather than explain everything directly
- visuals should feel like the brand and suggest **infinite possibility**

This means Oviond creative should feel cleaner, more evocative, and more expansive than standard SaaS stock-design patterns.

## Brand posture

Design should feel:
- modern
- digital
- confident
- spacious
- restrained
- intentional
- commercially credible
- not cold, but not playful-for-the-sake-of-it

Avoid turning the brand into either:
- generic B2B corporate office visuals
- chaotic, over-styled startup art

## Tagline / product framing

The guide shows the line:
- `Client reporting for your marketing agency.`

Use this as an anchor for product truth: agency-focused reporting, clear utility, and direct relevance to client reporting.

## Logo system

### Approved logo directions referenced in the guide

The brand guide explicitly shows:
- refreshed Oviond logo
- original logo comparison
- stacked logo applications
- horizontal logo applications
- web logo usage

### Logo typography

The guide explicitly states:
- **logotype font:** `Inter Semibold`

Treat this as fixed for logo expression.

### Practical logo rules for design work

Because the source extract confirms variants but does not provide exhaustive lockup measurements in text, use these working rules:
- use the **horizontal logo** by default for banners, headers, ads, and website placements where width allows
- use the **stacked logo** where vertical balance or square formats benefit from it
- use the **web logo treatment** for site/product-adjacent usage when the composition mirrors web UI placement
- do not redraw, stretch, distort, or decorate the logo
- do not improvise alternate logo fonts
- when placing over gradients or color fields, preserve strong contrast and legibility

If later asset files from the original guide include stricter spacing/minimum-size rules, those should replace these practical defaults.

## Color system

The guide is very explicit about restraint.

### Usage philosophy

Source wording to preserve:
- `Colour to be used sparingly. Not every blue, every time.`
- `Black, white and strong digital blue are the primary.`
- `Secondary colours we use sparingly.`
- `Keep it light with white and highlight with MAIN BLUE.`

### Primary palette

Use these as the core of the system:
- **White** — primary base
- **Black** — primary anchor/contrast color
- **MAIN BLUE** — `#0000FF` (explicitly shown in the guide as the strong digital blue)

### Secondary palette

The guide shows these supporting colors:
- `#0A0B5C`
- `#5676FF`
- `#D5E4FF`

Use them sparingly as support colors, depth colors, or gradient stops.

### Palette behavior

- default to **light compositions** with white carrying most of the space
- use **MAIN BLUE** as the most obvious accent/highlight color
- use the secondary blues with restraint rather than all at once
- avoid muddy, overmixed, multicolor palettes
- do not turn every asset into a blue overload just because the palette contains multiple blues

## Gradients

The guide includes explicit gradient examples derived from the palette.

### Shown gradient pairings
- `#0A0B5C -> #0000FF`
- `#0000FF -> #5676FF`
- `#5676FF -> #D5E4FF`

### Gradient rules

- use gradients as a controlled brand device, not as wallpaper everywhere
- keep gradients clean and digital, not noisy or muddy
- gradients can work behind the logo when contrast is strong
- prefer simple, elegant gradient use over complex multi-stop spectacle

## Imagery and textures

This is one of the most important parts of the guide.

### Required imagery direction

Use imagery that is:
- abstract
- unspecific
- aesthetic-led
- emotionally suggestive
- brand-feeling first

### Avoid overreliance on
- office scenes
- meeting-room shots
- laptop-around-a-desk clichés
- “people smiling at charts” stock imagery
- literal explanation visuals when an evocative brand image would do the job better

### What “infinite possibility” should look like in practice

Aim for visuals that feel:
- open
- dimensional
- light-driven
- digitally polished
- cleanly composed
- slightly expansive or atmospheric

This does **not** mean fantasy art, random sci-fi, or visual chaos.

## Typography

### Confirmed from the guide
- `Inter Semibold` is the selected font for the logo type.

### Working typography direction for design outputs

The source extract does not provide a full type scale, so use this working interpretation:
- favor clean, modern sans-serif typography
- keep text minimal and direct in generated compositions
- prefer bold, high-legibility headlines over dense copy blocks
- if the output is image-generated, keep in-image text short and simple

If a fuller type spec is later supplied from the source pack, replace this working guidance with the official one.

## Composition

The guide’s color and imagery direction together imply a clean, spacious composition system.

Use these composition rules:
- keep layouts light and open
- allow white space to do real work
- use color accents deliberately rather than flooding the canvas
- anchor designs with one strong focal point
- let brand feeling come through by restraint, not clutter

## Channel/application notes

The guide includes examples/applications for:
- web
- social
- Facebook
- LinkedIn
- YouTube

Use these general interpretations:
- **Web:** clean, light, polished, product-adjacent, logo used with precision
- **Social:** stronger focal point and clearer message hierarchy, but still recognizably minimalist
- **Facebook/LinkedIn:** prioritize clarity and brand consistency over decorative experimentation
- **YouTube:** use the brand line and stronger framing, but keep within the same clean digital system

## What to avoid

Reject any output that drifts toward:
- generic office-stock SaaS visuals
- cluttered dashboards as decorative filler
- overpacked layouts
- random extra blues everywhere
- weak contrast on white
- gradients used as gimmicks
- unapproved logo alterations
- visual styles that feel too literal, too corporate, or too “template marketplace”

## Working hierarchy of truth

When building or reviewing Oviond creative, follow this order:
1. official brand guide source
2. this reference file
3. asset-specific references in this skill
4. task-specific brief

If a future version of the original guide provides more granular rules, update this file rather than working around it in prompts.
